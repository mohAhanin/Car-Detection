مستندات API - رزرو پارکینگ

توضیحات کلی

این API برای مدیریت رزرو جایگاه‌های پارکینگ و تحلیل داده‌های رزرو طراحی شده است. امکانات اصلی شامل:

  • دریافت اطلاعات جایگاه‌های پارکینگ: وضعیت رزرو (موجود، رزرو شده توسط کاربر، غیرقابل دسترس) و قیمت.
  • ثبت درخواست رزرو: ارسال درخواست رزرو و دریافت نتیجه.
  • مدت زمان رزرو: مشخص کردن مدت زمان رزرو برای کاربر.
  • تحلیل داده‌های رزرو: گزارش‌هایی از پرکاربردترین و کم‌کاربردترین جایگاه‌ها، روزها و ساعات شلوغ.

Endpointها

۱. دریافت اطلاعات جایگاه‌های پارکینگ (GET /parking-spaces/)

توضیح: این Endpoint برای دریافت اطلاعات تمام جایگاه‌های پارکینگ (تنها یک فضای پارکینگ وجود دارد) و وضعیت رزرو هر کدام استفاده می‌شود. وضعیت‌های ممکن شامل:

  • available: جایگاه خالی و قابل رزرو.
  • taken: جایگاه توسط کاربر دیگری رزرو شده است.
  • reserved-by-user: جایگاه توسط کاربر جاری رزرو شده است.
  • not-available: جایگاه به دلیل ساخت‌وساز غیرقابل دسترس است.

همچنین، قیمت هر جایگاه نیز در پاسخ بازگردانده می‌شود.

۲. ثبت درخواست رزرو (POST /reservations/)

توضیح: این Endpoint برای ثبت درخواست رزرو یک جایگاه پارکینگ استفاده می‌شود. کاربر باید شناسه جایگاه و مدت زمان رزرو را ارسال کند. نتیجه رزرو (موفقیت یا شکست) پس از پردازش بازگردانده می‌شود. در صورت موفقیت، جایگاه برای مدت زمان مشخص‌شده رزرو می‌شود.

۳. دریافت مدت زمان رزرو (GET /reservations/{id}/duration/)

توضیح: این Endpoint برای دریافت مدت زمان رزرو یک جایگاه خاص استفاده می‌شود. شناسه جایگاه (id) باید در URL مشخص شود. مدت زمان رزرو بر حسب دقیقه بازگردانده می‌شود.

۴. تحلیل داده‌های رزرو (GET /reservations/analysis/)

توضیح: این Endpoint برای دریافت گزارش‌های تحلیلی از رزروها استفاده می‌شود. گزارش‌ها شامل:

  • پرکاربردترین و کم‌کاربردترین جایگاه‌ها: در هفته، ماه و سال.
  • روزهای هفته، هفته‌های ماه و ماه‌های سال: با بیشترین و کمترین رزرو.
  • ساعات شلوغ روز: برای رزرو.

هر گزارش شامل تاریخ دقیق (روز، ماه، سال) و تعداد رزروها است. برای مثال، برای هر هفته، تاریخ شروع و پایان هفته همراه با ماه و سال بازگردانده می‌شود.

پارامترهای ورودی

۱. ثبت درخواست رزرو (POST /reservations/)

پارامتر نوع داده اجباری؟ توضیح
lot-id string بله شناسه جایگاه پارکینگ.
duration number بله مدت زمان رزرو (بر حسب دقیقه).

نمونه درخواست‌ها

۱. ثبت درخواست رزرو (POST /reservations/)

    {
      "lot-id": "lot-001",
      "duration": 120
    }
