مستندات API - اطلاعات پارکینگ

توضیحات کلی

این API برای مدیریت اطلاعات پارکینگ‌ها و جایگاه‌های پارکینگ طراحی شده است. امکانات اصلی شامل:

  • ایجاد یا حذف فضای پارکینگ: تنها یک فضای پارکینگ می‌تواند وجود داشته باشد.
  • افزودن یا حذف جایگاه‌های پارکینگ: هر جایگاه پارکینگ یک شیء با مشخصات منحصر به فرد است.
  • دریافت اطلاعات کامل فضای پارکینگ: شامل لیستی از جایگاه‌های پارکینگ با جزئیات هر کدام.

هر جایگاه پارکینگ دارای فیلدهای زیر است:

  • id: شناسه منحصر به فرد جایگاه.
  • availability: وضعیت دسترسی (در دسترس یا به دلیل ساخت‌وساز غیرقابل دسترس).
  • position: موقعیت جایگاه در فضای پارکینگ (با فرض اینکه گوشه بالا-چپ فضای پارکینگ مختصات (۰, ۰) است).

Endpointها

۱. ایجاد فضای پارکینگ (POST /parking-space/)

توضیح: این Endpoint برای ایجاد یک فضای پارکینگ جدید استفاده می‌شود. تنها یک فضای پارکینگ می‌تواند وجود داشته باشد. اگر فضای پارکینگ از قبل وجود داشته باشد، این درخواست با خطا مواجه می‌شود.

۲. حذف فضای پارکینگ (DELETE /parking-space/)

توضیح: این Endpoint برای حذف فضای پارکینگ موجود استفاده می‌شود. اگر فضای پارکینگ وجود نداشته باشد، این درخواست با خطا مواجه می‌شود.

۳. افزودن جایگاه پارکینگ (POST /parking-space/lots/)

توضیح: این Endpoint برای افزودن یک جایگاه پارکینگ به فضای پارکینگ موجود استفاده می‌شود. هر جایگاه پارکینگ یک شیء با مشخصات منحصر به فرد است.

۴. حذف جایگاه پارکینگ (DELETE /parking-space/lots/{id}/)

توضیح: این Endpoint برای حذف یک جایگاه پارکینگ خاص از فضای پارکینگ استفاده می‌شود. شناسه (id) جایگاه پارکینگ باید در URL مشخص شود.

۵. دریافت اطلاعات فضای پارکینگ (GET /parking-space/)

توضیح: این Endpoint برای دریافت اطلاعات کامل فضای پارکینگ، شامل لیستی از جایگاه‌های پارکینگ با جزئیات هر کدام، استفاده می‌شود.

پارامترهای ورودی

۱. ایجاد فضای پارکینگ (POST /parking-space/)

پارامتر نوع داده اجباری؟ توضیح
width number بله عرض فضای پارکینگ (بر حسب واحد).
height number بله ارتفاع فضای پارکینگ (بر حسب واحد).

۲. افزودن جایگاه پارکینگ (POST /parking-space/lots/)

پارامتر نوع داده اجباری؟ توضیح
id string بله شناسه منحصر به فرد جایگاه پارکینگ.
availability boolean بله وضعیت دسترسی (true برای در دسترس، false برای غیرقابل دسترس).
position object بله موقعیت جایگاه پارکینگ (با مختصات x و y).

نمونه درخواست‌ها

۱. ایجاد فضای پارکینگ (POST /parking-space/)

    {
      "width": 100,
      "height": 50
    }

۲. افزودن جایگاه پارکینگ (POST /parking-space/lots/)

    {
      "id": "lot-001",
      "availability": true,
      "position": {
        "x": 10,
        "y": 20
      }
    }

نمونه پاسخ‌ها

۱. دریافت اطلاعات فضای پارکینگ (GET /parking-space/)

    {
      "width": 100,
      "height": 50,
      "lots": [
        {
          "id": "lot-001",
          "availability": true,
          "position": {
            "x": 10,
            "y": 20
          }
        },
        {
          "id": "lot-002",
          "availability": false,
          "position": {
            "x": 30,
            "y": 40
          }
        }
      ]
    }
